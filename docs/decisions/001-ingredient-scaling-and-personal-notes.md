# 001 — 재료 계산 & 개인 메모 기능 설계

날짜: 2026-06-14

## 배경

레시피 상세 페이지(`/recipes/[id]`)의 인분 스케일링 기능에서 "1.7스푼" 같은 실제 계량 불가능한 값이 나오는 문제. 아울러 사용자가 자신의 취향에 맞게 재료를 조정하고 메모를 남기는 기능도 함께 설계.

---

## 결정 1: 인분 선택기

**결정:** 고정 버튼 [1, 2, 3, 4] → +/- 버튼으로 교체, 범위 1~10인분

**이유:** 5인 이상 가족/파티 요리에 4인분 고정이 부족. 직접 입력은 모바일에서 키보드 UX가 불편.

---

## 결정 2: 단위별 반올림

현재 `Math.round(scaledAmount * 100) / 100`으로 소수 둘째 자리까지 표시하는 것을 단위별로 교체.

### 스푼류 (스푼, 큰술, 작은술, T, t)
- 0.5 단위로 반올림
- ½, ¼, ¾ 분수 표기로 출력 (소수 표기 금지)
- 예: 1.7스푼 → 1½스푼

### g / ml
구간별 반올림:
| 범위 | 반올림 단위 |
|------|------------|
| ≤ 10 | 1 |
| 10 ~ 100 | 5 |
| ≥ 100 | 10 |

**이유:** 소량(소금 3g → 5g)에서 고정 5단위 반올림 시 오차 67%. 구간별로 처리해야 맛에 영향 없음.

### 개수 단위 (개, 장, 줄기, 쪽, 알 등)
- `Math.ceil` (올림)
- **이유:** 재료가 모자라면 요리 자체가 안 되므로, 부족한 것보다 넉넉한 게 낫다.

### 텍스트 단위 (적당히, 약간, 조금 등)
- 그대로 유지, 스케일링 없음

---

## 결정 3: 개인 메모 & 재료 조정 기능

### 저장소
- **DB 저장** (localStorage 아님)
- 현재는 단일 사용자 앱이므로 사용자 구분 불필요. 추후 로그인 기능 추가 시 user_id 컬럼 연결.

### 조정 형식
- **퍼센트 조정** (절대값 아님)
- 인분 수 스케일링 후 퍼센트 적용: `scaled_amount * (1 + adjustment_pct / 100)`
- 범위: -80% ~ +100%, 단계: 10%

### UI 흐름
1. 재료 패널 우측 상단에 "내 취향으로 조정" 버튼
2. 클릭 시 편집 모드 진입:
   - 각 재료 옆에 -/+ 버튼 (10% 단계)
   - 하단에 자유 텍스트 메모 입력창
3. 조정값이 있는 상태로 일반 모드 복귀 시:
   - 상단에 배너 표시: "내 취향으로 조정됨"
   - 배너에 "원래 값으로 되돌리기" 버튼

## 구현 순서 (권장)

1. **Phase 1**: 인분 선택기 +/- 버튼 + 단위별 반올림 (프론트엔드만)
2. **Phase 2**: DB 스키마 추가 (recipe_personal_notes 테이블) + 개인 메모/조정 기능
