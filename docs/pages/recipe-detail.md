# `/recipes/[id]` — 레시피 상세

## 역할
개별 레시피의 재료와 조리법을 표시. 인분 수(1~4)를 선택하면 재료 용량을 클라이언트에서 자동 계산해 보여준다.

## 초기 로딩
페이지 진입 시 `$effect`가 `GET /api/recipes/{id}` 호출 → 성공 시 레시피 렌더, 실패 시 에러 박스 표시.

## 사용자 액션

| 액션 | 트리거 | API 호출 | 성공 | 실패 |
|------|--------|----------|------|------|
| 페이지 진입 | `$effect` 마운트 | `GET /api/recipes/{id}` | 레시피 데이터 렌더 | 인라인 에러 박스 표시 |
| 인분 버튼 (1·2·3·4) | `click` | 없음 | 재료 용량 재계산 (클라이언트) | — |
| 홈으로 돌아가기 | `click` (a 태그) | 없음 | `/` 로 이동 | — |

## 에러 케이스

| 상황 | 에러 메시지 | 표시 방식 |
|------|------------|-----------|
| API 호출 실패 (네트워크·서버) | `"레시피 조회 실패"` (api.ts 고정) 또는 서버 응답 텍스트 | 인라인 에러 박스 (빨간 배경) |
| 잘못된 ID (백엔드) | `"잘못된 ID"` (400) | 인라인 에러 박스 |
| 레시피 없음 (백엔드) | DB 에러 메시지 (500) | 인라인 에러 박스 |

## 상태

| 상태 | 초기값 | 설명 |
|------|--------|------|
| `recipe` | `null` | 로드 전 null → 스피너 표시 |
| `servingSize` | `1` | 현재 인분 수 |
| `error` | `''` | 에러 메시지 (비어있으면 에러 없음) |
| `activeIngredients` | `[]` | `recipe` × `servingSize` 파생값 |

## 인분 계산 공식
`최종 용량 = 기본 용량 × (1 + (인원수 - 1) × scaling_factor)`  
`scaling_factor`가 없으면 1.0 적용.

## 연관 API
- `GET /api/recipes/{id}` → `RecipeHandler.Get`
  - 400: "잘못된 ID" (id가 숫자가 아닐 때)
  - 500: DB 에러 문자열

## 연관 컴포넌트
- `$lib/api/recipes.ts` — `getRecipe()`
